## Task 0 (Prerequisite): Verify Phase 3 Manual Unity Editor Steps

**What**: **CRITIQUE MITIGATION** -- Phase 3 had 4 manual Unity Editor steps that cannot be done via code. If these are not complete, `_eliteDrumsPrefab` will be null and gameplay will silently fail to load any track.

**Action**:
1. Verify `EliteDrumsTrackPlayer` prefab exists in the Unity project (check Assets/Prefabs/Gameplay/)
2. Open Gameplay.unity scene in Unity Editor and verify `GameManager._eliteDrumsPrefab` field is assigned (not null)
3. Verify the prefab has `EliteDrumsPlayer` component with all required serialized fields wired:
   - `_fretArray` (EliteDrumsFretArray)
   - `_kickFretFlash` (KickFretFlash)
   - `_hatPedalIndicator` (if implemented)
4. If any are missing: complete the manual editor steps from Phase 3 Plan 03-03 before proceeding

**Verify**: GameManager inspector shows a valid prefab reference for `_eliteDrumsPrefab`.

**Done**: All Phase 3 manual Unity Editor steps confirmed complete. Prefab exists and is wired.

## Task 1: Verify Song Selection Flow

**What**: Verify the complete song selection experience for Elite Drums players.

**Action**:
1. **Music Library**: Confirm difficulty rings show Elite Drums availability for songs with PART ELITE DRUMS charts
2. **Game Mode Selection**: Select EliteDrums game mode from profile setup
3. **Difficulty Select**:
   - Confirm `Instrument.EliteDrums` appears as a selectable instrument
   - Confirm fallback instruments (FourLaneDrums, ProDrums, FiveLaneDrums) also available
   - Select Elite Drums instrument and verify difficulty selection works
4. **Binding Dialog**: Verify the Elite Drums binding dialog launches correctly from profile setup
5. **Edge case**: Select a song that does NOT have PART ELITE DRUMS -- verify fallback to 4L/5L works gracefully

**Verify**: Song can be selected and game begins loading with EliteDrums game mode.

**Done**: Song selection flow works end-to-end for Elite Drums with no exceptions.

## Task 2: Verify Full Gameplay Flow

**What**: Play through a song with Elite Drums and verify all gameplay elements function correctly.

**Action**:
1. **Song Loading**: Verify song loads without errors, track initializes with 10-lane highway
2. **Note Rendering**: Confirm notes scroll correctly on all 8 note lanes
3. **Input Registration**: Verify MIDI drum kit inputs register and trigger fret animations
4. **Hit/Miss Scoring**: Confirm hits and misses are scored correctly
5. **Star Power**: Verify star power activates (double kick or designated activation method)
6. **Solo Sections**: Verify solo box appears and scores solo percentage
7. **Combo Meter**: Verify combo increments on hits, resets on miss, multiplier increases
8. **HUD Elements**: Verify score box, progress bar, and all HUD elements display correctly
9. **Dynamics**: Verify accent/ghost notes render with visual differentiation
10. **Hi-hat States**: Verify open/closed hi-hat notes display correctly
11. **CRITIQUE MITIGATION -- Replay**: Record an Elite Drums gameplay session, then play back the replay. Verify:
    - Replay loads without errors
    - Input actions replay correctly (EliteDrumsAction enum values round-trip)
    - Score/stats match the original run
    - EliteDrumsPlayer.ConstructReplayData() produces valid replay data

**Verify**: Complete song playthrough with no crashes or visual glitches. Replay round-trip verified.

**Done**: Full gameplay loop works: notes render, inputs register, scoring works, HUD elements display.

## Task 3: Verify Score Screen & Fix Issues

**What**: Verify the score screen displays correct results and fix any issues found during verification.

**Action**:
1. **Score Screen**: After completing a song, verify:
   - Score card uses drums card prefab (DrumsStats)
   - Total score, accuracy, note streak displayed
   - Modifier icons show (if engine preset has modifiers enabled)
   - Star rating calculated
2. **Fix Issues**: For any bugs, crashes, or visual glitches found in Tasks 1-2:
   - Document the issue with file path and reproduction steps
   - Implement a fix
   - Re-verify after fix
3. **YARG.Core Tests**: Run unit tests to confirm no regressions:
   ```bash
   cd YARG.Core && dotnet test YARG.Core.UnitTests/YARG.Core.UnitTests.csproj
   ```

**Verify**: Score screen displays, tests pass, no open issues.

**Done**: Elite Drums end-to-end flow is verified complete from song selection through score screen.
