## Task 2: Create EliteDrumsColors Color Provider

**Action instructions:**

1. Read `YARG.Core/YARG.Core/Game/Presets/ColorProfile.cs` to understand the `IFretColorProvider` interface:
   ```csharp
   public interface IFretColorProvider
   {
       public Color GetFretColor(int index);
       public Color GetFretInnerColor(int index);
       public Color GetParticleColor(int index);
   }
   ```

2. Read `YARG.Core/YARG.Core/Game/Presets/ColorProfile.FourLaneDrums.cs` as the reference pattern. Note how it:
   - Defines individual Color fields per pad
   - Implements `IFretColorProvider` with index-based switches
   - Has `GetNoteColor`, `GetNoteStarPowerColor`, `GetActivationNoteColor`, `GetMetalColor`
   - Has a `Miss` color field
   - Has `GetKickFretColor()` and related kick colors

3. Create `YARG.Core/YARG.Core/Game/Presets/ColorProfile.EliteDrums.cs` as a partial class of `ColorProfile`. Define `EliteDrumsColors` class implementing `IFretColorProvider`.

   **CRITICAL (critique fix): Use 1-BASED indices for IFretColorProvider methods.**
   `FretArray.InitializeColor()` passes `i + 1` to color provider methods (where `i` is the 0-based fret index). Kick frets use index 0. This matches the FourLaneDrums convention.

   Index mapping for `GetFretColor(int)`, `GetFretInnerColor(int)`, `GetParticleColor(int)`:
   - 0 = Kick (Orange) — used by kick fret initialization
   - 1 = HiHat (Yellow) — fret 0 gets `i + 1 = 1`
   - 2 = LeftCrash (Blue)
   - 3 = Snare (Red)
   - 4 = Tom1 (Yellow)
   - 5 = Tom2 (Blue)
   - 6 = Tom3 (Green)
   - 7 = Ride (Blue)
   - 8 = RightCrash (Green) — fret 7 gets `i + 1 = 8`

   For `GetNoteColor(int)` used by the note element, use 0-based lane indices (0=HiHat, 7=RightCrash) since note elements manage their own indexing. Document both conventions clearly in the code.

   Use the same Color values from FourLaneDrums where applicable (Red, Yellow, Blue, Green, Orange). Include:
   - `GetFretColor(int)`, `GetFretInnerColor(int)`, `GetParticleColor(int)` (IFretColorProvider)
   - `GetNoteColor(int)`, `GetNoteStarPowerColor(int)`, `GetActivationNoteColor(int)`
   - `GetMetalColor(bool isStarPowerActive)`
   - `Miss` color field
   - Kick fret color methods (follow FourLaneDrums pattern)

4. Read `YARG.Core/YARG.Core/Game/Presets/ColorProfile.cs` again. Add a new field:
   ```csharp

   public EliteDrumsColors EliteDrums;
   ```
   Place it after the existing `FiveLaneDrums` field.

## Task 3: Fix FretArray PlayMissAnimation Bug

**Action instructions:**

1. Read `Assets/Script/Gameplay/Visuals/Fret/FretArray.cs`. Find the `PlayMissAnimation` method (around line 190).

2. Find the bounds check: `if (0 <= index && index <= _frets.Count)`. The `<=` should be `<` to prevent IndexOutOfRangeException. With 8 frets, `_frets.Count` is 8, and `index <= 8` would allow accessing `_frets[8]` which is out of bounds.

3. Fix: Change `index <= _frets.Count` to `index < _frets.Count`.

4. While in the file, verify the fret scaling formula works at 8 lanes:
   - Scale = `(TRACK_WIDTH / WIDTH_NUMERATOR) / (FretCount / WIDTH_DENOMINATOR)` = `(2/2) / (8/5)` = `0.625`
   - This is acceptable (ProDrums split view uses 0.714 at 7 lanes)
   - No changes needed to the formula

5. Check if there are any other similar off-by-one bugs in FretArray (other methods using `<= Count` instead of `< Count`). Fix any found.

**Verify:**
Search for all uses of `_frets.Count` in FretArray.cs and verify correct bounds checking.

**Done when:** `PlayMissAnimation` bounds check uses `<` instead of `<=`. No other off-by-one bugs remain. 8-lane fret scaling verified correct.
