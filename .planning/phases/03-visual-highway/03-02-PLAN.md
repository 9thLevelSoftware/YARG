### Architecture (from spec)
```
EliteDrumsPlayer : TrackPlayer<DrumsEngine, DrumNote>
├── Engine: DrumsEngine + InstrumentTrack<DrumNote>
├── Visual: _eliteTrack (InstrumentTrack<EliteDrumNote>) + correlation map
├── Notes: EliteDrumsNoteElement : NoteElement<DrumNote, EliteDrumsPlayer>
└── HatPedal: Visual-only indicator
```

### Critical implementation details
- **1-based lane indices**: GetElementX(lane, 8) with lane 1-8 aligns with FretArray fret 0-7
- **Input interception**: Override InterceptInput() to capture EliteDrumsAction BEFORE engine converts to DrumsAction
- **Correlation**: Prefer Path A (YARG.Core origin field) over Path B (parallel iteration)
- **Flam partner DrumNotes**: Hide visually, show flam indicator on real note only
- **Hat pedal**: State machine reading EliteDrumNote HatPedal events (Stomp/Splash/InvisibleTerminator)

**Required implementations (from TrackPlayer<> abstract methods):**

1. `GetNotes(SongChart chart)` — Return the correct `InstrumentDifficulty<DrumNote>` from the downcharted drum track. Follow DrumsPlayer's pattern: determine if 5-lane or 4-lane based on chart, get the appropriate track. Also store `chart.EliteDrums` as `_eliteTrack` for visual use.

2. `CreateEngine()` — Copy from DrumsPlayer. Use `DrumsEngineParameters.DrumMode.ProFourLane`, create `YargDrumsEngine` with `isEliteDrums: true`. Subscribe to engine events: OnNoteHit, OnNoteMissed, OnOverhit, OnPadHit, OnSustainEnd, OnStarPowerPhraseHit, OnStarPowerPhraseMissed, OnStarPowerStatus, OnSoloStart, OnSoloEnd.

3. `InitializeSpawnedNote(IPoolable poolable, DrumNote note)` — Cast poolable to `EliteDrumsNoteElement`. **Add runtime type check before cast (critique fix):** `if (poolable is not EliteDrumsNoteElement element) { YargLogger.LogError("NotePool prefab has wrong NoteElement type. Expected EliteDrumsNoteElement."); return; }`. Look up the corresponding EliteDrumNote using the correlation map. Set `ElitePadInfo` on the note element. Call the note element's initialization.

**Correlation mapping (build during initialization):**

Try Path A first:
1. Check if `DrumNote` has an `EliteDrumOrigin` field (YARG.Core change)
2. If available, use it directly in `InitializeSpawnedNote`
3. If not available (YARG.Core change blocked), fall back to Path B:
   - During init, iterate `_eliteTrack` and build `Dictionary<uint, List<EliteDrumNote>>` keyed by tick
   - In `InitializeSpawnedNote`, look up by `note.Tick` and match by pad mapping rules
   - Handle flam partners: if a DrumNote at tick T has no matching EliteDrumNote (it's a flam ghost partner), mark it for visual hiding

**For Path B implementation, the correlation matching:**
```csharp
// For each DrumNote at tick T:
// 1. Get all EliteDrumNote at tick T
// 2. For each EliteDrumNote, check if its downchart pad matches the DrumNote pad
// 3. Mark matched pairs; unmatched DrumNotes are flam partners
```

**FretArray initialization (in FinishInitialization):**
- Set `FretCount = 8`
- Use `Player.ColorProfile.EliteDrums` as color provider (from Plan 03-01)
- Pass `swapSnareAndHiHat: false`, `swapCrashAndRide: false`
- Initialize kick fret with kick color from EliteDrumsColors

**Input interception:**
```csharp
public override bool InterceptInput(ref GameInput input)
{
    // If input is an EliteDrumsAction, record which Elite lane was hit
    // Store in _lastEliteActions queue (action, time) for OnPadHit to consume
    // Do NOT consume the input — let it pass through to the engine
    return false;

```

**OnPadHit handler:**
- When engine fires OnPadHit(DrumsAction), look up the _lastEliteActions queue
- Find the most recent EliteDrumsAction that corresponds to this DrumsAction
- Map to Elite lane index (0-7 for frets)
- Call FretArray.PlayHitAnimation(laneIndex) or PlayCymbalHitAnimation(laneIndex)

## Task 2: Create EliteDrumsNoteElement.cs

**Action instructions:**

Read these files thoroughly first:

- `Assets/Script/Gameplay/Visuals/TrackElements/Drums/FourLaneDrumsNoteElement.cs` — lane positioning + color

Create `Assets/Script/Gameplay/Visuals/TrackElements/EliteDrums/EliteDrumsNoteElement.cs`:

```
EliteDrumsNoteElement : NoteElement<DrumNote, EliteDrumsPlayer>, IThemeNoteCreator
```

**ElitePadInfo struct:**
```csharp
public struct ElitePadInfo
{
    public EliteDrumPad Pad;       // Which of the 10 pads
    public DrumNoteType Dynamics;  // Neutral, Accent, Ghost
    public EliteDrumsHatState HatState;  // Open, Closed, Indifferent
    public bool IsFlam;
    public bool IsFlamPartner;     // If true, this note should be visually hidden
    public bool IsCymbal;          // HiHat, LeftCrash, Ride, RightCrash
}
```

**SetThemeModels (IThemeNoteCreator):**
- Create 7 NoteGroup arrays: Normal, Cymbal, Kick, Accent, Ghost, CymbalAccent, CymbalGhost
- Follow exact pattern from DrumsNoteElement.SetThemeModels()

**InitializeElement:**
1. If `ElitePadInfo.IsFlamPartner`, disable the entire note element (hide it)
2. Map ElitePadInfo.Pad to lane index (1-based):
   ```
   HiHat=1, LeftCrash=2, Snare=3, Tom1=4, Tom2=5, Tom3=6, Ride=7, RightCrash=8
   Kick: position at kick fret location (not a lane)
   HatPedal: should not appear as a note (filtered earlier)
   ```
3. Position: `transform.localPosition = new Vector3(GetElementX(lane, 8), 0f, 0f) * LeftyFlipMultiplier;`
4. Select NoteGroup based on `GetNoteGroup(isCymbal)` (see below)
5. If hi-hat pad AND HatState == Open, enable the open hat indicator child object
6. If IsFlam, enable the flam indicator child object

**GetNoteGroup(bool isCymbal):**
```csharp
// Follow DrumsNoteElement pattern but using ElitePadInfo.Dynamics
if (ElitePadInfo.Dynamics == DrumNoteType.Accent)
    return isCymbal ? CymbalAccent : Accent;
if (ElitePadInfo.Dynamics == DrumNoteType.Ghost)
    return isCymbal ? CymbalGhost : Ghost;
return isCymbal ? Cymbal : Normal;
// Kick has its own group
```

**UpdateColor:**
- Get color from `Player.Player.ColorProfile.EliteDrums`
- Map ElitePadInfo.Pad to color index (0-based: HH=0, LCrash=1, Snare=2, T1=3, T2=4, T3=5, Ride=6, RCrash=7)
- Handle lefty flip: reverse the color index (7 - index)
- Handle star power: use SP color when active
- Handle activation notes: pulse effect synced to BeatEventHandler
- Handle miss: use Miss color when NoteRef.WasMissed

**HitNote:**
- Return note to pool immediately (drums have no sustains)
- Follow DrumsNoteElement.HitNote() pattern

**Hi-hat open indicator:**
- Add a serialized `_openHatIndicator` child Transform/GameObject
- Enabled only when `ElitePadInfo.Pad == HiHat && ElitePadInfo.HatState == Open`
- Visual: ring mesh or simple sprite positioned around the note gem

**Flam indicator:**
- Add a serialized `_flamIndicator` child Transform/GameObject
- Enabled only when `ElitePadInfo.IsFlam == true`
- Visual: smaller duplicate gem at slight +Z offset (grace note style)

**Verify:** All NoteGroup indices match the 7 ThemeNoteType values. Lane positioning uses 1-based indices. Color provider indices are 0-based. Open hat and flam indicators are conditional.

**Done when:** `EliteDrumsNoteElement` renders notes at correct 8-lane positions with per-pad colors, dynamics-based note groups, hi-hat open indicator, and flam indicator.

## Task 3: Create Hat Pedal Indicator Component

**Action instructions:**

Read `YARG.Core/YARG.Core/Chart/Notes/EliteDrumNote.cs` to understand:
- `EliteDrumPad.HatPedal` (value 0)
- `EliteDrumsHatPedalType`: Stomp, Splash, InvisibleTerminator
- Hat pedal notes are actual chart events in the EliteDrumNote track

Create `Assets/Script/Gameplay/Visuals/TrackElements/EliteDrums/HatPedalIndicator.cs`:

```csharp
public class HatPedalIndicator : MonoBehaviour
{
    // State machine for hat pedal
    public enum PedalState { Closed, Open }

    [SerializeField] private Transform _pedalMesh;      // The visual mesh/sprite
    [SerializeField] private float _closedYPosition;     // Y position when closed
    [SerializeField] private float _openYPosition;       // Y position when open
    [SerializeField] private float _transitionDuration;  // Animation speed

    private PedalState _currentState = PedalState.Closed;
    private List<(double time, EliteDrumsHatPedalType type)> _pedalEvents;
    private int _eventIndex;
}
```

**Initialization:**
1. Called by `EliteDrumsPlayer` during init
2. Receives the EliteDrumNote track
3. Scans for all notes where `Pad == EliteDrumPad.HatPedal`
4. Filters out `InvisibleTerminator` events (no visual)
5. Stores as sorted list of (time, type) pairs

**Update (called each frame by EliteDrumsPlayer):**
1. Advance `_eventIndex` as visual time passes
2. On `Stomp` event: transition to `Closed` with pulse animation
3. On `Splash` event: transition to `Closed` briefly, then `Open` with pulse animation
4. Between events: maintain current state
5. Animate `_pedalMesh` Y position between closed/open via DOTween or lerp

**Position:**
- Place adjacent to HiHat lane (lane 0 / fret position 0)
- X position: same as GetElementX(1, 8) with slight offset to the left
- Fixed Z position: at or near the strike line

**Visual design:**
- Simple animated mesh (foot pedal shape or just a rectangular indicator)
- Color matches HiHat lane color (Yellow)
- When open: indicator raised
- When closed: indicator lowered
- Pulse: brief scale animation on state change (DOTween)

**Verify:** State machine correctly processes Stomp → Closed, Splash → Closed → Open. InvisibleTerminator filtered. Animation runs smoothly.

**Done when:** HatPedalIndicator component exists, processes chart events, animates between open/closed states, and is positioned correctly near the hi-hat lane.
