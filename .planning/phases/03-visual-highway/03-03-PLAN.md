## Task 1: Create EliteDrums TrackPlayer Prefab

**Action instructions:**

NOTE: Prefab creation requires Unity Editor operations that code agents cannot fully automate. This task produces the CODE changes and documents the manual Unity Editor steps needed.

1. Read the structure of an existing drum prefab by examining how `_fourLaneDrumsPrefab` is referenced and used:
   - Read `Assets/Script/Gameplay/GameManager.Loading.cs` to understand how prefabs are instantiated
   - Read `Assets/Script/Gameplay/Player/DrumsPlayer.cs` serialized fields to understand what the prefab must wire

2. Create a documentation file at `Assets/Prefabs/Gameplay/EliteDrums/PREFAB_SETUP.md` (or include in summary) listing the exact manual Unity Editor steps:

   **Step-by-step prefab creation:**
   a. In Unity Editor, duplicate the FourLaneDrums TrackPlayer prefab
   b. Rename to `EliteDrumsTrackPlayer`
   c. On the root GameObject:
      - Remove the `DrumsPlayer` component
      - Add the `EliteDrumsPlayer` component
      - Wire all serialized fields (FretArray, KickFretFlash, etc.)
   d. FretArray configuration:
      - Duplicate existing frets to have 8 total (from 4)
      - Position them evenly across the track
      - `FretCount` will be set at runtime by EliteDrumsPlayer
   e. Note pool:
      - Create an `EliteDrumsNoteElement` prefab (duplicate a DrumsNoteElement prefab)
      - Replace the DrumsNoteElement script with EliteDrumsNoteElement
      - Add child objects for `_openHatIndicator` and `_flamIndicator`
      - Wire the note prefab to the note pool
   f. Hat pedal indicator:
      - Add a new child GameObject for HatPedalIndicator
      - Add the `HatPedalIndicator` component
      - Position near hi-hat lane (X matches lane 0 fret position)
      - Wire serialized fields (_pedalMesh, positions, transition duration)
   g. Save as prefab at `Assets/Prefabs/Gameplay/EliteDrums/EliteDrumsTrackPlayer.prefab`

3. Verify the EliteDrumsPlayer serialized fields match what the prefab needs to wire:
   - `_fretArray` (FretArray)
   - `_kickFretFlash` (KickFretFlash)
   - `_hatPedalIndicator` (HatPedalIndicator)
   - Any other fields from TrackPlayer base

**Verify:** Document is complete with all manual steps. All serialized fields on EliteDrumsPlayer are identified and documented for wiring.

## Task 3: End-to-End Visual Verification

**Action instructions:**

This task is a comprehensive code review and verification pass, NOT a manual play test.

1. **Trace the full initialization path:**
   - GameManager.Loading.cs instantiates `_eliteDrumsPrefab`
   - Gets `EliteDrumsPlayer` component via `GetComponent<TrackPlayer>()`
   - Calls `Initialize()` → `GetNotes()` → `CreateEngine()` → `FinishInitialization()`
   - Verify each step works for EliteDrumsPlayer

2. **Trace the note spawning path:**
   - `TrackPlayer.UpdateNotes()` iterates `Notes` (InstrumentTrack<DrumNote>)
   - Calls `SpawnNote()` → `NotePool.KeyedTakeWithoutEnabling()` → `InitializeSpawnedNote()`
   - Verify `InitializeSpawnedNote` correctly sets ElitePadInfo on the note element
   - Verify `EliteDrumsNoteElement.InitializeElement()` positions at correct lane

3. **Trace the hit feedback path:**
   - Engine fires `OnNoteHit(index, DrumNote)` → `EliteDrumsPlayer.OnNoteHit()`
   - Engine fires `OnPadHit(DrumsAction)` → `EliteDrumsPlayer.OnPadHit()`
   - Verify input interception records EliteDrumsAction
   - Verify fret animation uses correct Elite lane index

4. **Trace the visual update path:**
   - `UpdateVisuals()` → `UpdateFretArray()` → fret press states
   - Note elements update position (Z movement) via TrackElement base
   - Hat pedal indicator updates from chart events

5. **Check for missing references:**
   - All `VisualStyle.EliteDrums` switch cases handled (no throws)
   - `ColorProfile.EliteDrums` used in the right places
   - No stale references to `DrumsPlayer` in EliteDrums code paths

6. **Check edge cases:**
   - Empty EliteDrumNote track (guard in init)
   - Lefty flip (lane reversal works at 8 lanes)
   - Star power activation (visual swap on note elements)
   - Flam partner DrumNotes (hidden in visual)
   - Hat pedal InvisibleTerminator (filtered)

   ```bash

   ```
